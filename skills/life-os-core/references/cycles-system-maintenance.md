# Cycles

## 目的

Cycles 是 **Review, Learn & Plan** 模块，用于持续校准方向和执行，防止系统随着时间退化、偏离或堆积无效内容。

## 周期层级

```text
Annual Review
  -> DuoCycles
    -> Weekly Review
      -> Daily Entries
```

每个层级承担不同时间尺度的反馈。

## DuoCycles

DuoCycle 是两个月的中期执行与复盘周期：

- 一年形成六个周期。
- 周期开始时定义成功标准。
- 每期承诺 1–3 个重点。
- 周期中通过 Weekly Review 调整。
- 周期结束时评估结果、总结学习并进入下一轮。

个人 Life OS 默认不再设置 Monthly 和 Quarterly Review，减少重复复盘。工作环境要求季度节奏时可以保留 Quarterly Review。

## Annual Review

Annual Review 回看长期方向和系统整体，重点检查：

- Pillars 是否仍准确表达核心价值。
- Aspirations 是否仍值得投入。
- 过去一年的重点、取舍和学习。
- 下一年度需要延续、停止或重新设计的方向。

## Weekly Review

Weekly Review 是 DuoCycle 执行期的主要校准节奏。默认检查：

- Inbox 是否需要处理。
- 活跃 Goals、Projects 和 Routines 是否健康。
- 无 Goal 的 Project/Routine 独立原因是否仍成立。
- Waiting On 与跟进日是否需要更新。
- 下周重要 Actions 是否已有合理 DO Date。
- 当前工作是否仍服务于 DuoCycle 重点。

## Daily Entries

Daily Entry 承载轻量记录：

- Routine 打卡。
- 当日简短反思。
- 值得进入 Remembrance 的 Notable 内容。
- 必要的状态或数据记录。

## 数据与 Visualization

- 只追踪能够支持决策的数据。
- 图表是可选反馈工具。
- 睡眠、运动、情绪和深度工作等指标按当前目标选择，不作为统一必填项。
